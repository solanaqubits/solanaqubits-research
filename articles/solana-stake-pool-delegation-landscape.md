# Solana Stake Pool Delegation Landscape: A Practical Snapshot for Validators

Last updated: 2026-06-21

This research note is a practical, transparent snapshot of Solana stake pool delegation programs for validators, stakers, staking pool operators, and ecosystem contributors.

It does not make financial promises and does not imply endorsement, sponsorship, recommendation, or official partnership with any pool or program.

## Discussion

A short launch and discussion thread for this resource is available on X:

https://x.com/solanaqubits/status/2067259338364711235

## Methodology

The snapshot is based on Validators.app data, rounded for practical comparison. Stake, validator counts, and averages should be treated as directional rather than exact.

Difficulty is estimated for an SFDP validator with 0–5% validator commission and 0% Jito MEV commission.

Difficulty is subjective. It reflects practical fit, apparent eligibility path, and likely outreach effort, not a guarantee of acceptance or delegation.

## Difficulty scale

| Difficulty | Meaning |
| --- | --- |
| 1 | Most practical. Clear fit for an SFDP validator with matching criteria. |
| 2 | Practical target with good performance and clear application or eligibility path. |
| 3 | Moderate difficulty. Fit depends on region, mission, connectivity, or curated review. |
| 4 | Harder target. Usually requires stronger ranking, infrastructure fit, or elite recent performance. |
| 5 | Longer-term target. Economics, bonds, auctions, or strict commission/performance profiles dominate. |

## Most practical near-term targets

### Phase

Phase appears to be one of the most practical near-term targets for an SFDP validator that can show ecosystem contribution and remains below the stated stake threshold.

- Approx stake: 1.080M SOL
- Validators: 97
- Average per validator: 11.1k SOL
- Difficulty: 1
- Best fit: SFDP + <300k SOL + ecosystem contribution
- Links:
  - <https://phase.cc/apply>
  - <https://phase.cc/delegation>
  - <https://phase.cc/tools/sfdp>
  - <https://phase.cc/delegation/ips>
  - <https://phase.cc/blog/solana-foundation-delegation-program-data>
  - <https://phase.cc/delegation/validators>

### JPool

JPool is a practical target when a validator can satisfy the program requirements around bond/direct stake, performance, and contribution expectations.

- Approx stake: 1.289M SOL
- Validators: 202
- Average per validator: 6.4k SOL
- Difficulty: 2
- Best fit: practical target; bond/direct stake/contribution
- Links:
  - <https://docs.jpool.one/delegation-strategy/how-to-join-jpool-delegation-program.html>
  - <https://docs.jpool.one/delegation-strategy/inclusion-and-removal-criteria.html>
  - <https://app.jpool.one/validators>
  - <https://docs.google.com/forms/d/e/1FAIpQLSc7GX3g2womS6sGHNZU8cau4K5ZyEmlSZ0ZdrBpKhP86q11tQ/viewform>

### BlazeStake

BlazeStake is another realistic target for validators with clean operational performance. For validators above larger stake thresholds, a 0% validator commission and 0% MEV commission profile can be more competitive.

- Approx stake: 1.034M SOL
- Validators: 171
- Average per validator: 6.1k SOL
- Difficulty: 2
- Best fit: good target with clean performance; if stake is >50k, 0/0 is better
- Link:
  - <https://stake-docs.solblaze.org/protocol/delegation-strategy>

## Strategic mid-difficulty targets

### DynoSol

DynoSol looks more curated than purely formulaic. That can make it attractive, but also harder to evaluate without direct review and a clear validator story.

- Approx stake: 1.637M SOL
- Validators: 77
- Average per validator: 21.3k SOL
- Difficulty: 3
- Best fit: human/curated review; rules are less strictly formalized
- Links:
  - <https://docs.dynosol.io/>
  - <https://www.dynosol.io/#stats>

### DoubleZero

DoubleZero can be a strong strategic target for validators that meet network connectivity and external stake requirements, but it should not be treated as a generic application path.

- Approx stake: 5.174M SOL
- Validators: 377
- Average per validator: 13.7k SOL
- Difficulty: 3–4
- Best fit: strong target only if there is DZ connectivity + 10k external stake
- Link:
  - <https://doublezero.xyz/dzdp>
  - <https://doublezero.xyz/dzdp/calculator>

### Definity

Definity may be relevant when a validator fits the program’s regional mission and SFDP-oriented criteria. Fit matters more than broad outreach.

- Approx stake: 265k SOL
- Validators: 30
- Average per validator: 8.8k SOL
- Difficulty: 3 if region fits
- Best fit: requires SFDP + regional mission
- Link:
  - <https://definity.finance/whitelist-apply>

### JagPool

JagPool looks most relevant for validators aligned with its target regions or those able to meet the external stake path. Region fit is a key factor.

- Approx stake: 727k SOL
- Validators: 82
- Average per validator: 8.9k SOL
- Difficulty: 3 if region fits
- Best fit: LATAM/APAC/South Africa; SFDP or 40k external stake
- Links:
  - <https://docs.jagpool.xyz/DELEGATION-STRATEGY/Delegation-Criteria>
  - <https://docs.jagpool.xyz/DELEGATION-STRATEGY/Validator-Application-Process>

### Edgevana

Edgevana is a tighter fit for validators already using Edgevana infrastructure and competing for top rankings. Validator age and infrastructure path matter.

- Approx stake: 820k SOL

- Difficulty: 4
- Best fit: only Edgevana infrastructure + top 50 ranking; age tier is important
- Links:

### SolStrategies

SolStrategies appears to emphasize strong recent performance. A validator should focus on a clean 30-day score and measurable consistency before outreach.

- Approx stake: 597k SOL
- Validators: 70
- Average per validator: 8.5k SOL
- Difficulty: 4
- Best fit: main path is high 30-day StakeWiz score
- Link:

### Vault

Vault has a baseline path, but the strongest competitive profile appears to be closer to 0% validator commission and 0% MEV commission with elite performance.

- Approx stake: 1.327M SOL
- Validators: 168
- Average per validator: 7.9k SOL
- Difficulty: 4
- Best fit: baseline <5% commission; elite performance path is effectively 0/0

## New or active delegation opportunities to monitor

These items are useful to monitor because they expose public program criteria, dashboards, application/interest paths, or curated delegation mechanisms. Listing them here does not imply endorsement, partnership, approval, acceptance, or guaranteed delegation.

### Phase IPS / SFDP resources

- SFDP analytics: <https://phase.cc/tools/sfdp>
- IPS methodology: <https://phase.cc/delegation/ips>
- SFDP data update: <https://phase.cc/blog/solana-foundation-delegation-program-data>
- Validator dashboard: <https://phase.cc/delegation/validators>

Phase is useful both as a delegation program/resource and as a public source of SFDP and IPS analytics. According to the Phase IPS documentation, the Index Power Score uses the following components: Contributions — 30%, Sustainability — 30%, Phase Partner — 25%, and Infrastructure Diversity — 15%.

The contribution component describes ecosystem contributions such as infrastructure improvements, open-source tooling, educational work, and community initiatives. Validators should verify the current IPS methodology and review cadence directly with Phase documentation.

### Vault board-gated delegation buckets

- Source: <https://docs.thevault.finance/delegation/approved-validators#board-gated-delegation-buckets>

Vault documentation describes board-gated eligibility and bucket-based delegation opportunities. The docs describe a Direct Stake Leader Bucket at 40% and an Elite Performance Bucket at 50% of undirected stake allocation. According to the docs, allocation is handled by stakebot and updates every 3 hours.

Approval, eligibility, ranking, or board review does not guarantee allocation. Validators should verify current requirements, commission expectations, MEV expectations, uptime expectations, fee obligations, and bucket status with Vault sources.

### Marinade Select and SAM resources

- Marinade Select docs: <https://docs.marinade.finance/marinade-protocol/protocol-overview/marinade-select.md>
- Marinade Select announcement: <https://marinade.finance/blog/marinade-select-unlocking-institutional-staking-on-solana>
- Marinade institutions page: <https://marinade.finance/institutions>
- SAM resources: <https://docs.marinade.finance/marinade-protocol/protocol-overview/stake-auction-market/sam-resources.md>
- SAM stake matching: <https://docs.marinade.finance/marinade-protocol/protocol-overview/stake-auction-market/stake-matching.md>

Marinade Select is described by Marinade as a curated or vetted validator set for institutional staking. It is separate from the open Stake Auction Marketplace path and includes additional validator expectations such as identity verification, bonding, and ongoing performance standards. Acceptance, participation, or delegation is not implied by this research note.

SAM resources remain relevant Marinade context because they provide dashboards, APIs, auction data, validator bond tooling, and stake matching documentation for validators participating in or monitoring Marinade delegation mechanics.

### JPool Community Good validator stake application form

- Form: <https://docs.google.com/forms/d/e/1FAIpQLSc7GX3g2womS6sGHNZU8cau4K5ZyEmlSZ0ZdrBpKhP86q11tQ/viewform>

The form is titled “JPool Stake Application for Community Good Validators.” It asks validators and projects for public-good contribution details, project traction, open-source or free-license details, intended use of additional stake rewards, KPIs, and value for JPool.

This should be treated as an application or interest form only. Submitting an application does not guarantee review, approval, stake, or delegation.

## What validators should monitor

Validators evaluating stake pool or delegation opportunities should monitor:

- public validator scoring dashboards;
- stake pool eligibility criteria and source updates;
- application windows, interest forms, or validator onboarding paths;
- validator commission and MEV commission expectations;
- uptime, delinquency, and operational performance;
- performance ranking, vote credits, and vote latency where relevant;
- SFDP status and external stake composition;
- public contributions, open-source work, education, infrastructure work, and community initiatives;
- official documentation, governance, dashboard, and source updates.
